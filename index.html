Checklist for Web Design

HTML rules

  • Tags are always surrounded by angle brackets
  • Most tags come in pairs and surround the material they affect
  • The second tag always starts with a forward slash
  • When you start a tag within another tag,
    you have to close that inner tag before closing the outer tag
  • Many tags have can take arguments that change the tag's behavior

CSS rules

  • Any HTML element can be a CSS selector
  • A simple selector can have different classes
  • Classes may also be declared without an associated element
  • ID selectors target only one unique element at a time
  • You can assign values to properties of CSS classes and elements
  • A property is assigned to a selector in order to change its style

Technical and file management rules

  • Avoid extra long folder names and complicated folder structures but use well-named filenames instead
  • Use underscores between elements
  • Use hyphens between words within an element or capitalize the first letter of each word within an element
  • Elements should be ordered from general to specific detail of importance as much as possible
  • This is also true when elements include date and time stamps

Content

  • The content meets user needs, goals, and interests
  • The content is in a usable format, including headings, bulleted lists, tables, white space, or similar techniques is appropriate
  • The customer needs to get benefits from the content
  • Use attention grabbing headlines
  • Use consistent voice

Design foundations

  • Typography
    • Use Web fonts and Google Fonts
    • Use consistent styles
    • Make good use of white space
    • No more than 50-60 characters should be used per line
  • Color theory
    • Pages provide high contrast between text and background to make it easy to read
    • Text and link colors are clearly different from one another
    • The colors fit the theme
    • The pages use color consistently
  • Style tiles
    • Contain sample user interface elements
    • Define a color scheme
    • Show how typographical design will be used
    • Show the relationship between the client's brand and the proposed design
  • Grid
    • Grids provide organization and visual balance
    • Grid columns provide the structure for the content columns
    • Grids used in responsive web design usually consist of 12 or 24 columns
    • The horizontal space the grid takes up - the width - grows and shrinks with the available display space

Saving images correctly

  • Compress images with the right format at the lowest acceptable quality level
  • Look into using WebP for your image needs
  • Save your images with smaller sizes to improve your pages' load times
  • Investigate other interesting ways to get better compression, or transparency

Accessibility/usability

  • The background and text have appropriate contrast
  • The website loads within 3 seconds
  • The site logo is easy to find and links to the home page
  • The About Us, Contact Us and Home links are easy to find
  • The text on the page is easy to read
  • The design, layout and organization of the site are professional and consistent
  • The navigation is clear, well organized and does not overwhelm the user

Code Validation

  • Validate HTML and CSS
  • Validate Links
  • Can be read by different browsers
